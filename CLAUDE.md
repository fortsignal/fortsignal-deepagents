# DeepAgents Maintainer Rules

## Publishing to PyPI — STRICT RULES

**Never bump the version or publish for README, docs, or comment-only changes.**

Only publish when `fortsignal_deepagents/` has actual code changes.

### Correct release flow (code changes only):
```bash
# 1. Bump version in pyproject.toml manually (e.g. 0.1.6 → 0.1.7)
# 2. Commit and tag
git commit -m "Bump to 0.1.7 for PyPI publish"
git tag v0.1.7
git push && git push --tags

# 3. Build
python -m build

# 4. Publish (requires PyPI token)
python -m twine upload dist/fortsignal_deepagents-0.1.7* -u __token__ -p YOUR_PYPI_TOKEN
```

### README/docs only:
```bash
git commit -m "Update README"
git push
# STOP. Do not bump version or publish.
```

## Versioning
- Version lives in `pyproject.toml` under `[project] version`
- Every PyPI publish must have a corresponding git tag
- No automated CI/CD — all publishing is manual

## What lives here
- `fortsignal_deepagents/` — Python package source. Only changes here warrant a new version.
- `dist/` — build output, generated by `python -m build`. Not committed to git.
- `README.md` — public docs. Edit freely, push to GitHub, never triggers a publish.

## What does NOT live here
- The FortSignal API (fortsignal-api) — separate Cloudflare Workers project
- The TypeScript SDK (@fortsignal/sdk) — separate repo at fortsignal/fortsignal-sdk
